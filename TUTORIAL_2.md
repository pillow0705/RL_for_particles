# 模型架构升级：从 GNN 到 Transformer

## 为什么要替换 GNN？

原有架构使用图神经网络（GNN）编码已放置的粒子状态，存在几个明显问题：

**1. 训练无法真正并行**
GNN 需要邻接矩阵，而每个训练样本的粒子数不同，邻接矩阵大小也不同。训练时只能用 Python for 循环逐个计算图编码，256 个样本的 batch 实际上是串行的，GPU 利用率极低。

**2. 邻接矩阵维护复杂**
每放一个粒子就要增量更新邻接矩阵，代码复杂且容易出错。

**3. 感受野受限**
GNN 每层只能聚合直接相邻的粒子，需要多层才能感知全局结构。

## 训练时的 Batch 处理

```
Batch 内 B 个训练样本，粒子数各不相同：

  样本 1: N₁ = 12 个粒子    ████████████░░░░░░░  (padding)
  样本 2: N₂ = 19 个粒子    ███████████████████
  样本 3: N₃ =  7 个粒子    ███████░░░░░░░░░░░░  (padding)
  ...
  样本 B: Nᵦ = 15 个粒子    ███████████████░░░░  (padding)

                              ↑ max_N = 19 对齐 ↑

  padded  tokens : (B, max_N,   4)  ← padding 位置填零
  pad_mask       : (B, max_N    )   ← True = 忽略此位置
  [CLS] + tokens : (B, max_N+1, d)

  Transformer 通过 src_key_padding_mask 自动跳过 padding 位置，
  CLS 输出只汇聚真实粒子的信息，不受 padding 污染。

  最终: CLS → (B, 128)   一次前向，B 个样本真正并行 ✓
```

## 新架构设计思路

### 核心想法：粒子作为 Token

把已放置的 N 个粒子看作一个序列，每个粒子是一个 token，特征为：

```
[(x-cx)*0.5,  (y-cy)*0.5,  (z-cz)*0.5,  r/r_max]
 ↑ 质心相对坐标，PBC wrap 后 ×0.5 缩放      ↑ 归一化半径
```

坐标与候选点特征使用**完全相同的参考系**（均以当前堆积质心为原点，PBC wrap，缩放 0.5），使得 CLS 与候选点做点积时两者处于同一坐标空间，点积结果具有几何意义。

**不需要位置编码**：NLP 中的位置编码是为了补充 token 的顺序信息，而粒子本身的 xyz 坐标已经包含了完整的空间位置信息，粒子之间也没有顺序关系，加位置编码反而引入错误的偏置。

**直觉**：CLS 是"当前状态需要什么样的粒子"的方向向量，候选点嵌入是"我是什么样的粒子"的自我描述，点积高说明这个候选点最符合当前需求。这比原来 MLP FusionDecoder 更简洁，也更有几何意义。

---

## 同步修复的问题

**采集与训练的 temperature 不一致**

采集时：
```python
probs = softmax(scores / T)   # 用了 temperature
```
训练时原来：
```python
log_probs = log_softmax(scores)   # T 硬编码为 1
```

修复后训练时统一除以 temperature，保证策略梯度计算的 log π(a|s) 与采集分布一致。

## 架构对比

| | 原 GNN 架构 | 新 Transformer 架构 |
|---|---|---|
| 粒子编码 | GNN + 邻接矩阵 | Self-Attention，无需邻接矩阵 |
| 训练并行 | for 循环，串行 | 真正 batch，并行 |
| 候选点打分 | MLP FusionDecoder | CLS 点积，直接高效 |
| 代码复杂度 | 高（邻接矩阵维护） | 低 |
| 感受野 | 局部邻居 | 全局 |

## 关键设计巧思总结

以下是本项目中提出的核心设计思想，按类别整理，供后续写作参考。

---

### 一、模型架构

**① 粒子天然不需要位置编码**

NLP 中 Transformer 引入位置编码是为了区分 token 的先后顺序。但粒子堆积问题中，每个粒子的 `(x, y, z)` 坐标本身已经是完整的空间位置信息，粒子之间也没有任何顺序关系（第 10 个放入的粒子不比第 5 个放入的更"重要"）。加入位置编码反而会引入错误的序列偏置。这使得该问题天然适合"纯空间特征 + Transformer"的组合。

**② CLS token 作为堆积状态的压缩摘要**

借鉴 BERT 的 [CLS] 设计，在粒子序列前插入一个可学习 token。经过多层 Self-Attention 后，[CLS] 汇聚了所有粒子的全局信息，输出一个固定维度的向量，回答"当前堆积整体是什么状态、需要什么样的粒子"。这比池化（mean/max）更灵活，比引入解码器更简洁。

**③ 用点积打分替代 MLP 解码器**

原架构用 MLP FusionDecoder 将全局状态与每个候选点拼接后打分，计算量随候选点数量线性增长且耦合度高。新方案将其分解为两个独立嵌入的点积：

```
score[i] = CLS · cand_emb[i]
```

CLS 是"需求向量"，候选点嵌入是"供给向量"，点积衡量两者的方向一致性。这不仅计算更高效（O(N·d) vs O(N·d²)），也赋予了打分过程几何意义。

**④ 粒子坐标与候选点坐标使用同一参考系**

粒子 token 和候选点特征必须在同一坐标空间下，点积才有意义。二者统一使用以当前堆积质心为原点、经 PBC wrap、缩放 0.5 的相对坐标：

```
rel = (pos - COM_pbc)  →  PBC_wrap  →  × 0.5
```

这同时带来平移不变性：不论堆积整体漂移到盒子哪个位置，相对结构不变，模型泛化更好。

### 二、训练策略

**⑤ Advantage 过滤：丢弃中间、保留两端**

REINFORCE 中，归一化 advantage 接近 0 的样本对策略梯度贡献极小，但仍参与反向传播，引入无效噪声。过滤掉 `|advantage|` 最小的 30%，只保留表现明显好于或差于平均水平的样本，使梯度方向更纯净：

```
保留条件: |advantage| ≥ quantile(|advantage|, 0.3)
```

这类似于优先经验回放（Prioritized Experience Replay）的思想，但实现更简单直接。

**⑥ 模型退步时回滚 + 学习率减半**

每轮训练完成后备份当前模型权重。若下一轮采集到的 `phi_max` 相比上一轮下降超过阈值（默认 0.01），则认为发生了策略退步，自动恢复到上一轮权重，并将学习率缩小 0.5 倍：

```
if phi_max < prev_phi_max - rollback_tol:
    恢复权重  +  lr × 0.5
```

比固定 lr schedule 更自适应：只在真正退步时才缩小步长，避免过早收敛。

**⑦ 只在创历史最优时保存模型**

每轮训练后，仅当本轮 `phi_max` 超过历史最优记录时才保存 checkpoint 和对应堆积文件。避免用差模型覆盖好模型，确保磁盘上的 `_best.pth` 始终对应真正最优的策略。

**⑧ 采集与训练 temperature 保持一致**

策略梯度的正确性要求：训练时计算的 `log π(a|s)` 必须与采集时实际执行的分布完全一致。采集时用了温度 T 的 softmax，训练时 `log_softmax` 也必须同样除以 T，否则二者描述的是不同的策略，梯度计算失效。

### 三、工程设计

**⑨ 实验目录以时间戳命名**

多人协作时，若实验目录按序号命名（`run_001`、`run_002`），不同成员在本地并行实验再 push 时会产生命名冲突。改为以启动时刻的时间戳命名（`20240315_143022`），全局唯一，无需协调。

**⑩ 候选点集合增量维护**

候选点不在每步重新从零生成，而是增量更新：放入新粒子后，只过滤掉与新粒子碰撞的旧候选点，再基于包含新粒子的三元组生成新候选点。维护四个映射结构（`candidate_set`、`triplet_set`、`cand_to_triplets`、`cand_counter`），将每步复杂度从 O(N³) 降至近似 O(N²)。
